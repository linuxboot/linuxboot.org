Additional Reasons for LinuxBoot Just in case the front page did not convince you… You can use any filesystem that Linux supports, not just FAT. You can implement boot policies using normal Linux applications, e.g. shell scripts or binaries, rather than manipulating opaque NVRAM variables. You can run Linux applications straight from the ROM. You can avoid legacy partitions entirely and use LVM for flexible volume management. You can build it yourself and verify that the reproducible build matches what others have built to ensure that the firmware is clean. [Read More]
